# GraphRAG TMDB Demo — Project Prompt

## Purpose

You are an expert software engineer. Build a demo application called **CineGraph** that
demonstrates why Knowledge Graph RAG (GraphRAG) is fundamentally better than standard
vector RAG for relationship-heavy, multi-hop questions over movie data.

The goal is not just to build a working app — it is to make the *architectural insight*
visible. The user should be able to ask a question, see the Cypher query that was
generated, watch the graph render in real time, and get an answer that a standard RAG
system provably cannot produce from chunks alone.

---

## Tech Stack

| Layer       | Technology                                           |
|-------------|------------------------------------------------------|
| Backend     | Python 3.11+, FastAPI                                |
| LLM         | OpenAI GPT-4o (chat completions)                     |
| Graph DB    | Neo4j AuraDB (free tier)                             |
| Frontend    | Streamlit                                            |
| Graph viz   | streamlit-agraph                                     |
| HTTP client | httpx (Streamlit → FastAPI calls)                    |

**No vector database. No embeddings. No hybrid retrieval.** All retrieval is done
exclusively through Neo4j Cypher queries generated by the LLM. This is intentional —
the demo's thesis is that for relational queries, graph traversal alone outperforms
vector similarity search.

The entire project is Python-only. No Node.js, no npm, no JSX. A single
`pip install -r requirements.txt` installs everything.

## Dataset

Use the **TMDB Movie dataset** loaded via Neo4j's built-in movie graph
(`:play movie-graph` in Neo4j Browser), extended with a TMDB API fetch for richer data
if needed.

The graph schema must include at minimum:

```
(:Movie  {title, year, tagline, revenue, rating})
(:Person {name, born})
(:Genre  {name})
(:Studio {name})

(:Person)-[:ACTED_IN  {roles}]->(:Movie)
(:Person)-[:DIRECTED       ]->(:Movie)
(:Person)-[:WROTE          ]->(:Movie)
(:Movie )-[:IN_GENRE       ]->(:Genre)
(:Movie )-[:PRODUCED_BY    ]->(:Studio)
```

Load the base dataset using the official Neo4j movie graph Cypher seed script.
Optionally augment with the TMDB API (free key) to add Genre and Studio nodes.

## Core Architecture

### Backend — FastAPI

FastAPI runs as a separate process. Streamlit calls it over localhost HTTP. This
separation keeps LLM and Neo4j logic cleanly isolated from the UI layer and makes
the backend independently testable.

**`POST /api/query`**
- Accepts: `{ "question": "string" }`
- Internally:
  1. Sends the question + Neo4j schema description to GPT-4o with a system prompt
     instructing it to return only a valid Cypher query (no markdown, no explanation)
  2. Executes the Cypher query against Neo4j AuraDB
  3. Sends the raw query results back to GPT-4o to synthesize a natural language answer
  4. Returns:
     ```json
     {
       "answer":     "string",
       "cypher":     "string",
       "hop_count":  2,
       "graph_data": {
         "nodes": [{"id": "...", "label": "Movie|Person|Genre", "properties": {...}}],
         "edges": [{"source": "...", "target": "...", "type": "ACTED_IN"}]
       },
       "meta": {
         "node_count": 14,
         "edge_count": 11,
         "query_ms":   87
       }
     }
     ```
- `graph_data` contains the subgraph traversed, formatted for streamlit-agraph

**`GET /api/schema`**
- Returns the current Neo4j graph schema (node labels, relationship types, property
  keys) so Streamlit can display it in the sidebar and the LLM prompt stays accurate

**`GET /api/examples`**
- Returns the hardcoded list of example questions (see below)

**`GET /health`**
- Simple liveness check; Streamlit polls this on startup to confirm the backend is up

### LLM Prompt Design (Critical)

**Cypher generation system prompt** must:
- Provide the full graph schema with node labels, relationship types, and sample
  property values
- Instruct the model to output ONLY valid Cypher — no markdown fences, no explanation,
  no preamble
- Require `RETURN` clauses to include full node and relationship objects (not just
  scalar properties) so the backend can build `graph_data` for visualization
- Include 3–5 few-shot examples covering single-hop, multi-hop, aggregation, and
  shortest-path queries
- For shortest-path queries, use `shortestPath()` — do NOT use APOC procedures
  (AuraDB free tier does not guarantee APOC availability)
- Cap results: add `LIMIT 100` to all queries

**Answer synthesis prompt** must:
- Receive the original question, the generated Cypher, and the raw result rows
- Respond conversationally, citing specific movie titles and actor names from the results
- End with one sentence explaining why this query required graph traversal and could
  not be answered reliably from text chunks

### Backend — Key Design Constraints
- All credentials loaded from `.env` via `python-dotenv` — never hardcoded
- Cypher execution wrapped in try/except; errors returned as structured JSON with a
  `"error"` key and a human-readable `"detail"` — never let FastAPI return a raw 500
- CORS configured to allow `http://localhost:8501` (Streamlit default port)
- Async Neo4j driver (`neo4j` package, async session) so FastAPI stays non-blocking

## Frontend — Streamlit (`frontend/app.py`)

The entire frontend lives in a single `app.py` file. Use `st.set_page_config` with
`layout="wide"` to get a full-width canvas.

### Layout — three columns

```
st.columns([1, 2, 1])
  col1  →  controls + examples   (weight 1)
  col2  →  graph canvas          (weight 2)
  col3  →  answer + cypher       (weight 1)
```

**Column 1 — Controls**
- "CineGraph" title (`st.title`) with subtitle `st.caption("GraphRAG Demo · TMDB + Neo4j")`
- `st.text_area` for natural language question input
- "Ask" button (`st.button`, `type="primary"`)
- `st.divider()`
- "Try an example" subheader; render each example question as its own `st.button`.
  Clicking one writes the question into `st.session_state` and triggers an immediate
  query submission — no second click required
- `st.expander("Graph Schema")` — fetches `/api/schema` once on load (cache with
  `st.cache_data(ttl=300)`), displays node labels and relationship types as a formatted
  markdown list
- `st.info()` callout box: "**Why Graph RAG?**" followed by 2–3 sentences explaining
  that graph traversal resolves multi-hop relational questions that vector similarity
  search cannot answer regardless of chunk size

**Column 2 — Graph Canvas**
- Use `streamlit-agraph` (`from streamlit_agraph import agraph, Node, Edge, Config`)
- Node color by label:
  - `Movie`  → `#E8593C`  (coral)
  - `Person` → `#7F77DD`  (purple)
  - `Genre`  → `#1D9E75`  (teal)
  - `Studio` → `#BA7517`  (amber)
- Node size: 25 for Movie, 20 for Person, 15 for Genre/Studio
- Node label: `title` property for Movie nodes, `name` for all others
- Edge label: relationship type (e.g. `ACTED_IN`, `DIRECTED`)
- `Config` settings: `height=580`, `width="100%"`, `directed=True`,
  `physics=True` (force-directed layout), `hierarchical=False`
- While a query is running, show `st.spinner("Traversing the graph…")` in this column
- When no query has been run yet, show `st.info("Ask a question or pick an example to see the graph")`
- After a successful query, call `agraph(nodes=..., edges=..., config=config)`
- If the query returns zero nodes, show `st.warning("Query returned no results. Try rephrasing.")`

**Column 3 — Answer + Cypher**
- If the backend returned an error key, show `st.error(detail)` and stop rendering
- Otherwise, in order:
  1. `st.subheader("Answer")`
  2. `st.markdown(answer)`
  3. `st.divider()`
  4. `st.subheader("Generated Cypher")`
     `st.code(cypher, language="cypher")`
  5. `st.divider()`
  6. Three `st.metric` widgets in `st.columns(3)`:
     - "Nodes" → `meta.node_count`
     - "Edges" → `meta.edge_count`
     - "Time"  → `f"{meta.query_ms} ms"`
  7. `st.caption(f"This query traversed {hop_count} hop(s) across the graph.")`

### Streamlit State Management
- Use `st.session_state` to persist results across reruns
- Keys to manage: `question` (str), `result` (full API response dict), `loading` (bool)
- Clicking an example button sets `st.session_state.question` and sets a flag that
  causes the query to fire immediately on the next rerun without pressing "Ask" again
- On query submission: call `httpx.post("http://localhost:8000/api/query", ...)`,
  store the full response dict in `st.session_state.result`

### Startup Health Check
On first load, call `GET http://localhost:8000/health` with a 2-second timeout.
If it fails or times out, show:
```python
st.error("⚠️ Backend not reachable. Start it with: uvicorn backend.main:app --reload")
st.stop()
```
This prevents the UI from rendering in a broken state.

## Example Questions (The Demo Script)

These must be rendered as clickable buttons in column 1. Each is chosen because it
requires multi-hop traversal that vector similarity search cannot reliably replicate.

| # | Button label | Why it needs graph traversal |
|---|---|---|
| 1 | `[1 hop] Which movies did Tom Hanks act in?` | Warm-up — single edge traversal |
| 2 | `[2 hops] Which actors worked with both Tom Hanks and Meg Ryan?` | Set intersection across two paths |
| 3 | `[2 hops] Which directors have worked with Keanu Reeves more than once?` | Edge-count aggregation |
| 4 | `[3 hops] Which actor has appeared in the most genres?` | Cross-entity aggregation |
| 5 | `[4+ hops] How is Kevin Bacon connected to Tom Hanks?` | Shortest path — the showpiece |
| 6 | `[3 hops] Which movies share the director and genre of The Matrix?` | Multi-condition graph join |
| 7 | `[3 hops + exclusion] Actors in Nolan films who never co-starred together?` | Exclusion reasoning — impossible for vector RAG |

## Quality Constraints

- Backend starts with one command: `uvicorn backend.main:app --reload` from project root
- Frontend starts with one command: `streamlit run frontend/app.py` from project root
- No Docker required — all local dev works with plain Python
- `.env.example` must document every variable with an inline comment
- All FastAPI errors caught and returned as `{"error": true, "detail": "..."}` JSON
- Streamlit health check on load — never render the UI against an unreachable backend
- `agraph` must not crash on empty node/edge lists — guard with a zero-result warning
- Schema fetch cached with `st.cache_data(ttl=300)` — never re-query on every rerun
- Node/edge results capped at 100 in the Cypher query — prevents the agraph canvas
  from becoming unreadable on broad queries

## What Success Looks Like

A first-time viewer watching a screen recording of this app should:

1. See a clean, three-column Streamlit interface that immediately communicates its
   purpose — no clutter, no setup friction visible
2. Click "[4+ hops] How is Kevin Bacon connected to Tom Hanks?" and watch the graph
   render a force-directed path connecting two distant nodes through intermediate actors
3. Read the generated Cypher in the right column and understand it as graph traversal —
   the query literally walks edges, it does not search text
4. Read the LLM's answer naming the specific intermediate actors and movies on the path
5. Intuitively grasp that no amount of chunking, embedding, and cosine similarity
   would reliably reconstruct this multi-hop relational chain

That moment of insight — that is the demo.
